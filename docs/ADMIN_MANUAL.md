## 3. 직원관리

### 3-1. 엑셀 일괄 업로드

1. **엑셀 업로드** 버튼 클릭
2. 엑셀 파일 선택 (확장자 `.xlsx` 또는 `.xls`)
3. 자동으로 직원 정보 등록 + 기존 직원 정보 갱신

### 엑셀 컬럼 형식

| 컬럼명 | 필수 | 비고 |
|---|---|---|
| 이름 | ✅ | 신청자 식별 키 |
| 이메일 | ✅ | 회사 메일 (연차 알림 수신) |
| 부서/팀 | 선택 | |
| 팀장이메일 | 선택 | 임원이면 비워둠 |
| 임원여부(Y/N) | 선택 | Y면 팀장 CC 미발송 |
| 입사일 | ✅ | 자동 연차 계산 기준 |
| 재입사일 | 선택 | **입력 시 입사일 대신 사용** (재입사 = 근속 새로 시작) |
| 총연차(자동계산) | 선택 | 임의 입력값 우선 |
| 이미 사용한 일수(선반영) | 선택 | 시스템 도입 전 사용분 |

### 3-2. 개별 추가·수정

- **+ 개별 추가**: 단일 직원 추가
- 직원 행의 **수정** 버튼: 입력한 정보 변경
  - 입사일 변경 → 자동 연차 재계산
  - **이미 사용한 일수**: 시스템 도입 전 회사에서 쓴 연차를 입력 (잔여에서 차감)

### 3-3. 정년 촉진계약자·특수 케이스
직원 수정 모달에서 **총연차** 칸에 직접 일수 입력 → 자동계산 무시 (수동 우선).

## 4. 신청내역

### 4-1. 상태 종류
- **대기** (pending): 직원이 신청 후 미처리
- **승인** (approved/auto_approved): 처리 완료
- **반려** (rejected): 거부됨

### 4-2. 액션 버튼

| 상태 | 가능 액션 |
|---|---|
| 대기 | 승인 / 반려 / 수정 / 삭제 |
| 승인 | 승인취소 / 수정 / 삭제 |
| 반려 | 사유 보기 / 수정 / 삭제 |

### 4-3. 권장 운영 흐름
- 잘못 신청된 건은 **반려 → 직원 재신청** (감사 로그 그대로 남음)
- **삭제·승인취소·직접 수정**은 예외 보정용으로만 사용

### 4-4. 자주 발생하는 케이스

**Q. 직원이 잘못 신청해서 다시 받고 싶을 때**
A. 반려 → 사유 입력 → 직원이 재신청

**Q. 승인을 잘못 눌렀을 때**
A. 승인취소 → 대기 상태로 복귀 → 다시 처리

**Q. 사유만 수정하고 싶을 때**
A. 수정 → 사유 변경 → 저장

## 5. 사용촉진 (근로기준법 제61조)

### 5-1. 법정 통지 시기
- **1차 통지**: 매년 7/1 ~ 7/10 (직원에게 사용 시기 지정 회신 요구, 10일 내)
- **2차 통지**: 매년 10/20 ~ 10/31 (1차 미회신자에게 회사가 사용 시기 지정 통지)
- 통지 후 미사용 연차는 보상(수당) 지급 의무 면제
- 통지 기록은 **3년 보존**

### 5-2. 발송 절차

1. **사용촉진 탭** 진입
2. **통지 단계** 선택 (1차 / 2차)
3. 2차일 경우 **회사 지정 사용 시기** 입력 (예: `2026-12-22 ~ 2026-12-31 (잔여 일수 일괄)`)
4. 대상 직원 체크박스 선택 또는 **대상자 전체 발송**
5. 확인 다이얼로그 → 발송

### 5-3. 대상자 자동 추출 기준
- 잔여 연차 5일 이상
- 임원 포함 (단, 김홍정 부사장 제외)

## 6. 출퇴근 대조 (SECOM)

### 6-1. 업로드 절차

1. SECOM에서 월간 출퇴근 엑셀 다운로드 (.xls/.xlsx)
2. **출퇴근 대조 탭** → **SECOM 엑셀 업로드** 클릭
3. 업로드 즉시 자동 파싱 + 기존 신청 데이터와 대조

### 6-2. 누적 저장
- 업로드 시 Gist에 월별 누적 저장
- 같은 월 재업로드 → 해당 월만 덮어쓰기 (다른 월 보존)
- 페이지 새로고침해도 데이터 유지
- **13개월 보존** (그 이전 자동 삭제)

### 6-3. 표시 룰

#### 출퇴근 시간 정규화
- **9시 이전 출근 → 9시로 인정**
- **9:30 이내 출근 → 9시로 인정**
- **18시 이후 퇴근 → 18시로 인정**
- 점심시간 11:45~12:45 자동 차감

#### 표기 규칙
- 7시간 30분 이상 → **`08:00`** 표기 + 정상 출근
- 미만 → 정확한 시간 표기 + 연차 사용 매칭

### 6-4. 판정 컬러

| 컬러 | 의미 |
|---|---|
| 🟢 녹색 (정상) | 정확한 출근 또는 정확한 휴가 매칭 |
| 🔵 파랑 (늦은 퇴근) | 1시간 늦게 퇴근 흡수 (예: 7시간 근무 → 반반차로 판정) |
| 🟡 노랑 (대기) | 신청 대기 상태 + 결근 |
| 🔴 빨강 (이상) | 미신청 결근, 신청 후 출근, 시간 불일치 |

### 6-5. 다운로드
**결과 엑셀 다운로드** 클릭 → 매월 정산용 자료로 활용

### 6-6. 김홍정 부사장 제외
김홍정 부사장은 SECOM 대조 대상에서 자동 제외 (그 외 임원은 포함).

- **연차 산정 방식**: **법정 자동**

## 8. 데이터 백업·복구

### 8-1. 자동 백업
- 모든 데이터는 GitHub Gist에 저장
- Gist는 **revision history 자동 보존**
- 실수 삭제 시 직전 버전으로 복구 가능

### 8-2. 수동 백업이 필요한 경우
1. https://gist.github.com (Salgi-bot 계정 — 김홍정 부사장 보유)
2. 해당 Gist 열기 → **Revisions** 탭
3. 직전 버전 raw 내용 복사 → 현재 파일 덮어쓰기

(이 작업은 김홍정 부사장에게 요청)

## 9. 자주 발생하는 문제

### Q1. 로그인이 안 됩니다
- 비밀번호 `0494` 정확히 입력 (대소문자 구분 없음)
- 그래도 안 되면 김홍정 부사장에게 문의 (PIN 변경됐을 수도)

### Q2. 직원이 신청했는데 메일이 안 와요
- 설정 탭 → **관리자 이메일** 정확한지 확인
- 스팸함 확인
- 직원 등록 시 이메일 정확한지 재확인

### Q3. 잔여 연차가 이상하게 나와요
- 직원 수정 모달 → **이미 사용한 일수**에 시스템 도입 전 사용분 입력
- 입사일·재입사일 정확한지 확인

### Q4. SECOM 업로드 후 이상치가 너무 많아요
- 김홍정 부사장 외 임원도 SECOM 대상 (정상 동작)
- "늦은 퇴근" 컬러 → 1시간 정도 야근은 흡수됨 (정상)
- "이상" 컬러만 검토하면 됨
